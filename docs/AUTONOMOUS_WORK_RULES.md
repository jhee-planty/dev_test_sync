# 자율 실행 절대 규칙 — NEVER STOP WORKING

## 핵심 원칙
**작업을 중지하지 마라. 사용자가 "멈춰"라고 명시적으로 말하기 전까지 계속 진행한다.**

## 금지 행위
1. 결과 보고 후 멈추고 사용자 응답 대기 ❌
2. "다음 단계를 진행할까요?" 같은 질문 ❌
3. "말씀해주세요" 등 사용자 지시 대기 문구 ❌
4. 상태 요약만 하고 실행을 멈추는 것 ❌

## 필수 행위
1. 결과 도착 → 즉시 분석 → 즉시 다음 조치 실행 ✅
2. 결과 미도착 → 다른 생산적 작업 병행 + 주기적 폴링 ✅
3. 판단 필요 시 → 최선의 판단으로 실행, 결과에 근거 기록 ✅
4. 한 작업 완료 → 바로 다음 작업으로 이동 ✅

## 폴링 규칙
- 스케줄러 불필요. 메인 세션이 작업 흐름 안에서 직접 폴링
- git pull → results/ 스캔 → 새 결과 있으면 처리 → 없으면 다른 작업 → 반복
- stall_count ≥ 6 (30분) → 해당 서비스 STALLED, 다음 서비스로 진행

## 작업 우선순위 (결과 대기 중 할 일)
1. 다른 서비스의 DB 조치/템플릿 개선
2. Group D 서비스 HAR 캡처 준비 (service_config.py 등록)
3. 기존 서비스 경고 개선 분석
4. 파이프라인 문서 업데이트
