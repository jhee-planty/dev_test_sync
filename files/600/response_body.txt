=== Diagnosis ===

The APF intercept IS firing on POST /ai/v2/generation (URL pattern matched, response is being substituted by the v2 envelope). However the v2 envelope's response headers do NOT include `Access-Control-Allow-Credentials: true`, which is mandatory because the gamma frontend issues this fetch with `credentials: 'include'` (cookies/auth needed for ai.api.gamma.app subdomain).

Browser drops the response at the CORS layer BEFORE the SSE body is parsed. The frontend never sees a single `event: content` chunk — it just gets `TypeError: Failed to fetch`. Hence:
- No spinner forever (the fetch errors out fast, ~57ms)
- No warning text rendered (response body never reaches React)
- Outline area shows empty card "1" — initial state, never replaced

=== Fix needed in v2 envelope (id=16) ===

Add to the response headers section (in addition to existing Content-Type / Content-Length):
  Access-Control-Allow-Origin: https://gamma.app
  Access-Control-Allow-Credentials: true
  Access-Control-Allow-Headers: content-type, accept, aijsx-traceparent, ...
  Access-Control-Expose-Headers: *

Reference: ai.api.gamma.app real upstream returns these CORS headers on its real responses. Our v2 envelope omits them, causing the browser to refuse the response.

This is NOT the same regression as #599 mistral (where the envelope itself was malformed). Here the envelope SHAPE is correct (event:content + data + event:done + data:stop per #598 ground truth), but the HTTP RESPONSE HEADERS are missing the CORS preflight-pair headers required for cross-origin SSE with credentials.
