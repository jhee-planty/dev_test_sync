## Schema captured (from benign sample, see response_body.txt)
- Transport: text/event-stream (SSE)
- Frame: "event: <type>\ndata: <text>\n\n" repeated, terminated by "event: done"
- Observed event types: youChatToken (data is JSON {"youChatToken": "..."}), done.
- Anonymous responses prompt user to log in; authenticated responses presumably
  contain richer event types (citations, workflow steps, search results).

## What dev should do next (schema-driven)
The capture proves we can replace the body if the engine switches from
RST-on-detect to body-substitution. Suggested injection (kept simple, you.com
parses chunk-by-chunk):

  HTTP/1.1 200 OK
  Content-Type: text/event-stream;charset=utf-8

  event: youChatToken
  data: {"youChatToken":"[보안 경고] 입력하신 내용에 개인정보(주민등록번호)가 포함되어 차단되었습니다.\n조직 보안 정책에 따라 외부 AI 서비스로의 전송이 거부됩니다."}

  event: done
  data: blocked-by-apf

This will render as the chat answer text inside the same response bubble
that currently shows the generic error.
