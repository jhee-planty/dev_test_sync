This suggests the engine's bundled close frame (0x88 02 03 E8) inside the

Possible engine-side adjustments:

  C) Use a longer/fragmented WS data frame so client must finish parsing
     before encountering the close opcode 0x88.
  D) Alternative: drop the close frame entirely and let the existing
     server-issued reconnect cycle handle it (the client already retries
     on its own — see "Retrying, retry attempt: 0").
