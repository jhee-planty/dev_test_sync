== Recommendation ==
v9 array→object change does NOT achieve the warning-bubble UX win the test was designed to validate.
Two paths forward for engine team:
  A) Keep v9 wrapper format ({...}) but drop the ?batch=1 path discrimination — route the request to non-batch tRPC handler so tRPC client unmarshals a single result-object correctly.
  B) Revert to v7 array wrapper ([{...}]) and instead investigate why v7 "Unknown error" surfaced — there may be a missing top-level field (e.g., shape, data, meta) that tRPC-batch expects inside each array element. The v9 error message ("Cannot convert undefined or null to object") points at exactly such a missing-field probe.
Either way, mission-critical revert (v9 → v7 emergency) is NOT required — both versions are equally safe (TRPCClientError + blank UI), and v9 is no worse than v7 from a security standpoint.
