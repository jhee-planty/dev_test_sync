=== Root cause hypothesis ===
The new template id=80 (SSE envelope) is not being applied for the you.com
service. Possibilities:
  (a) Template registration didn't pick up id=80 for service=you on the engine
  (b) you.com uses HTTP/2 or HTTP/3 multiplexing, and the engine's substitution
      hook fires before headers can be injected, falling through to TCP close
  (c) you.com's request URL pattern (POST /api/streamingSearch) isn't matching
      the template URL pattern in the engine config

Recommendation: verify on engine side that:
  - Template id=80 is loaded for the youchat domain
  - The match pattern includes POST /api/streamingSearch
  - The engine handler can write SSE headers on the negotiated HTTP version
