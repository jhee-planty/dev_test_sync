**Analysis Date:** 2026-04-13

**Purpose:** Structured review for multi-participant discussion  

---

## Executive Summary

This document provides a detailed structural analysis of 5 interconnected APF (AI Prompt Filter) skills. These skills collectively implement a 7+ phase workflow for capturing AI service prompts, blocking sensitive ones, and displaying user-friendly warnings. The analysis reveals:

- **Total Content:** ~1,500 lines across 5 SKILL.md files
- **Cross-references:** Heavy interconnection between all skills
- **Key Architecture:** Pipeline orchestration (genai-apf-pipeline) → specialized phases → sub-skills for implementation
- **Notable Gaps:** References to external files (guidelines.md, phase-specific reference files) are essential but not analyzed here

- **Line 185:** `BLOCKED_ONLY 판정은 존재하지 않는다` (BLOCKED_ONLY does NOT exist)

- **genai-warning-pipeline SKILL.md (line 185):** Also claims BLOCKED_ONLY does NOT exist

#### 4. **Inconsistent Terminology**

- **Line 131-154:** Warning display is the ONLY success criterion, period
- **Interpretation:** The first statement (line 34) suggests a fallback mode; the second (line 131) is absolute

- **Lines 292-332:** Describes L1→L2→L3 escalation with Scheduled Task handling
- **Assertion (line 324):** "L3는 메인 세션에서만 실행 가능하다" (L3 only works in main session)

#### 6. **Duplicate "Service Goals" Section**
- **Line 131-154:** Here in genai-apf-pipeline
- **genai-warning-pipeline line 99-121:** Nearly identical section in both skills
- **Risk:** Updates to one may not sync to the other

#### 7. **Missing Context Recovery Detail**
- **Line 411-440:** Context recovery procedure detailed for genai-apf-pipeline
- **genai-warning-pipeline (line 305-325):** Also has context recovery, nearly identical
- **Issue:** Two master orchestrators defining nearly identical recovery—maintenance burden

#### 8. **Vague Regression Testing Gate**
- **Line 193-195:** Regression gate required at Phase 3 and Phase 7
- **Detail missing:** Which services must be retested? All VERIFIED/DONE? How often?

#### 9. **Unresolved BLOCKED_ONLY Paradox**
- **This skill (line 185):** "BLOCKED_ONLY 판정은 존재하지 않는다"

### Recommendations for Discussion

1. **BLOCKED_ONLY Policy Clarification:** Is it forbidden, deprecated, or valid? If valid, how does it coexist with "all alternatives must be tried"?
2. **Phase-Skip Procedure:** Document the exact conditions and steps for skipping to Phase 4
3. **Scheduled Task Escalation Ownership:** Clarify which session (main/Scheduled Task) owns the L3 escalation decision
4. **Duplicate Sections Consolidation:** Merge genai-apf-pipeline and genai-warning-pipeline context recovery into one

## Conclusion

These 5 skills form a complex, interconnected system for APF (AI Prompt Filter) warning delivery. The pipeline is well-architected for parallel dev↔test collaboration and experiential learning, but suffers from:

1. **Unresolved policy contradictions** (BLOCKED_ONLY, success definition)

3. **Duplicate master orchestrators** with overlapping logic
4. **Inconsistent phase numbering** between two orchestrators

1. Policy alignment (BLOCKED_ONLY, success criteria, "all alternatives vs. exclude impossible")
2. Making external references available and validated
3. Consolidating genai-apf-pipeline and genai-warning-pipeline into a single, non-overlapping architecture
4. Defining clear, measurable success criteria for each phase
