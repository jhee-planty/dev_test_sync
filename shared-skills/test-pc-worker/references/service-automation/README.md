# Service Automation Profile

서비스별로 "접속 → 프롬프트 입력 → 전송"까지의 자동화 경로를 기록한다.
test PC가 check-warning/check-block 수행 시 이 프로필을 먼저 참조한다.

## 프로필 작성 형식

```markdown
# {서비스명} — Automation Profile

## 기본 정보
- URL: {접속 URL}
- 로그인: {필수/불필요, 방법}
- API 도메인: {실제 프롬프트 전송 도메인}

## 프롬프트 입력까지 네비게이션
1. {URL 접속}
2. {버튼 클릭 / 페이지 이동}
3. {입력창 도달}

## 검증된 입력 방식
- 1순위: {SendKeys / clipboard / JS / CDP}
- 대안: {실패 시 시도할 방식}
- 알려진 제약: {React 거부 등}

## 검증된 PowerShell 명령어
{실제 성공한 명령어}

## 최종 업데이트
- 날짜: {YYYY-MM-DD}
- 결과: {성공/실패 + 비고}
```

## 프로필 생성/업데이트 시점

- **최초 생성**: dev PC가 웹 검색으로 사용법 파악 후 작성하여 전달
- **테스트 후 업데이트**: test PC가 성공/실패 경험으로 검증된 방식 기록
- **실패 시**: dev PC에 프로필 보완 요청 (웹 재검색 포함)

## 프로필이 없는 서비스

기본 절차로 진행: URL 접속 → 입력창 탐색 → SendKeys 시도.
성공하면 프로필을 새로 생성한다.
